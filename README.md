# レベル f(n) バースト

[English README is here / 英語版はこちら](./README.en.md)

数列を唱えると、呪文書（**Berlekamp–Massey のアルゴリズム**）がその数列の最小の線形漸化式を見破り、漸化式の軌道上のレベルを持つ敵をすべて撃ち抜く——そんなパズルゲームです。某RPGの「レベルを条件に全体へ作用する青魔法」へのオマージュですが、「倍数」の代わりに「あなたが設計した漸化式の軌道」で敵を狙います。

## ルール

- 数字をカンマ区切りで唱えると、最小の漸化式が自動で見破られ、**位数（漸化式の長さ）＝消費MP** になります。デタラメな数列ほど位数が上がって高くつき、構造を見抜いた人ほど安く倒せます。
- 魔法は詠唱を止めたあとも**漸化式のまま飛び続けます**。「5, 10, 15, 20」と唱えれば次は25——延長線上に仲間がいたら直撃し、回復に2MPかかります。
- 次の項が**分数**になると魔法は霧散し、**レベル99を超える**と場外へ消えます。これを逆手に取り、`3, 8, 13, 21` のように等差をわざと崩して係数を分数化させ、軌道を自壊させて仲間を避けるのが上級テクです。
- 唱えた項数が位数の2倍未満だと漸化式の解釈が複数残ります（**解釈複数**バッジ）。解釈が複数でも呪文書の出力は決定的なので、プレビューに表示される軌道がそのまま飛びます。位数×2項まで唱えると軌道が一意に確定します（**軌道確定**バッジ）。
- 残りMP以内で敵を全滅させればクリア。見本MP以下なら魔道士級です。
- 手作りの3面をクリアしたら「∞」ボタンからエンドレスモードへ。**4面以降は自動生成で無限に続きます**。面が進むほど呪文の種類・敵の数・仲間トラップが増え、MPの余裕が減っていきます。

## 中身のアルゴリズム

ゲームの審判はすべて Berlekamp–Massey が務めています。

- **MPコストの判定**: 入力された数列に対し、有理数体上（BigInt分数）で厳密に最小位数の漸化式を復元します。任意の数列は高々 ⌈N/2⌉ 程度の位数で表せてしまうため、「全敵レベルを列挙して1発で全滅」は必ず高コストになり、ゲームバランスが定理から自動的に生まれます。
- **軌道の延長**: 復元した漸化式で数列を延長します。項が非整数になった時点で停止（霧散）、絶対値が99を超えた時点で停止（場外）、周期数列は上限項数で停止します。
- **一意性バッジ**: 「項数 N ≧ 2 × 位数 L なら最小漸化式が一意」という定理をそのままUIにしています。例えば `1, 2, 4, 7` は N=4, L=3 で解が無数にあり（階差+1、トリボナッチ、その他）、バッジが琥珀色になります。
- **無限ステージ生成**: 4面以降は「解となる呪文（等差・等比・分数比ペア・単体スナイプ・偽装等差）を先に生成し、その軌道から敵を採取する」構成的生成です。想定解は審判と同じ `traj()` で実シミュレートして検証するため、解けない面は存在しません。ステージ番号でシードした決定論的乱数を使うので、同じ面番号はいつでも同じ問題になります（到達面はブラウザに保存されます）。

## 開発

- 単一の `index.html`（HTML + CSS + Vanilla JS）。外部依存・ビルド工程なし。
- 数値計算はすべて `BigInt` の分数で厳密に行っているため、浮動小数点誤差による誤判定はありません。
